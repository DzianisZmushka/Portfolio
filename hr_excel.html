The main customer requirement was to create an Excel-based timesheet product that generates planning sheets based on data extracted from SAP SuccessFactors (HCM), with the ability to retract data (change data in the system). This product was developed for the human resources department of a logistics company. Data extraction was performed using PowerQuery, which was connected to the data in SAP via REST API, after which data transformation and processing was also performed in PowerQuery. Finally, planning sheets were built using VBA based on the extracted data.

The main menu was developed as the basis of the product, which included five VBA macros. The first macro was responsible for collecting parameters entered by the user. The second macro triggered data extraction from SAP SuccessFactors based on the customized parameters. When the third macro was run, planning sheets were generated. After user input, a fourth macro was run to retract the data to SAP. The fifth macro was used to check the data entered by the user in accordance with the customized business rules.

To support the product by the user, an options page has been added to allow the user to customize the planning sheets.

To monitor employees by the manager, a page was developed where it was possible to see if each employee had completed the planning sheets.

For ease of filling data into the planning sheets, a page with a table of daily models was created so that the user could specify any name for each item.

On the technical side, about 20 SAP entities were used to create the model at the PowerQuery level. The number of reports depended on the time period parameters entered by the user (one report for each month), as well as additional pages for parameter entry and employee control.
